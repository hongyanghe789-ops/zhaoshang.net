## 一、功能概述

### 1.1 背景与目标

房源空置期是园区最大的收益损失来源。本功能实现**房源到期前90天自动触发预招商机制**，在租户确认不续租后，房源状态自动转为"即将空置（可预订）"，系统主动提示一键发布到招商网络平台，最大限度缩短空置期。

### 1.2 核心价值

- 提前90天介入，缩短空置周期
- 自动化流转，减少人工遗漏
- 与平台联动，扩大招商触达面

## 二、预招商触发流程

### 2.1 整体流程

```
合同到期前90天
     │
     ▼
系统自动检测
     │
     ├── 租户已确认续租 → 不触发，走续租流程
     │
     └── 租户未确认续租 / 确认不续租
              │
              ▼
         生成"预招商任务"
              │
              ├── 推送通知给房源负责人/招商经理
              │
              ├── 房源状态 → "即将空置（可预订）"
              │
              └── 弹出"一键发布"引导
                       │
                       ▼
                  发布到招商网络平台
                  （自动打上"近期腾退"标签）
```

### 2.2 触发条件

| 条件 | 说明 |
|------|------|
| 时间条件 | 合同到期日前90天（可在系统设置中调整：60/90/120天） |
| 状态条件 | 合同状态为"执行中" |
| 排除条件 | 租户已提交续租申请且已审批通过 |

### 2.3 预招商任务

每个触发生成一条预招商任务记录：

| 字段 | 说明 |
|------|------|
| 任务编号 | 系统自动生成 |
| 关联合同 | 触发该任务的到期合同 |
| 关联房源 | 合同绑定的房源 |
| 当前租户 | 合同签约客商 |
| 到期日期 | 合同到期日 |
| 任务状态 | 待处理 / 已发布 / 已成交 / 已续租（取消） |
| 负责人 | 房源负责人/自动分配的招商经理 |
| 创建时间 | 系统自动生成 |

### 2.4 房源状态变更

```
房源状态机扩展：

[在租] ──(到期前90天+不续租)──→ [即将空置（可预订）]
                                      │
                               ┌──────┴──────┐
                               ▼              ▼
                         [锁定/谈判中]    合同到期→[空置]
                         (新客户签约)
                               │
                               ▼
                           [在租](新合同)
```

**"即将空置（可预订）"状态说明**：
- 当前租户仍在使用，合同仍有效
- 新客户可预订该房源（预定后→锁定）
- 在剖面图中用特殊颜色标识（紫色+斜线底纹）
- 平台展示"近期腾退，可预订"标签

## 三、一键发布引导

### 3.1 触发方式

预招商任务创建后，在以下位置展示引导：

| 位置 | 形式 |
|------|------|
| 消息通知 | 推送通知："房源A栋-301即将到期，建议提前发布招商" |
| 房源详情页 | 顶部黄色横幅提示："合同将于XX到期，[一键发布到平台]" |
| 预招商任务列表 | 操作列"一键发布"按钮 |

### 3.2 一键发布

点击"一键发布"后：
1. 自动携带房源信息打开载体发布页面
2. 自动勾选"近期腾退"标签
3. 自动填充预计可入驻日期（合同到期日+7天）
4. 用户确认/补充信息后发布

## 四、预招商任务管理

### 4.1 任务列表

**入口**：载体管理 → 预招商任务（新增菜单项）

**列表字段**：任务编号、房源名称、当前租户、到期日期、剩余天数、发布状态、任务状态、负责人、操作

**筛选条件**：园区、建筑、任务状态、到期时间范围

### 4.2 任务状态流转

```
[待处理] ──→ [已发布]（一键发布到平台）
   │              │
   │              ├──→ [已成交]（新客户签约）
   │              │
   │              └──→ [已续租]（原租户续租，取消预招商）
   │
   └──→ [已续租]（原租户续租，取消预招商）
```

## 五、系统配置

**入口**：系统设置 → 载体管理 → 预招商配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 提前触发天数 | 合同到期前N天触发 | 90天 |
| 自动发布 | 触发后是否自动发布到平台（无需人工确认） | 关闭 |
| 通知对象 | 预招商任务通知的接收人 | 房源负责人+部门主管 |
| 通知渠道 | 站内信/短信/UL推送 | 站内信+UL推送 |

---

## 六、数据权限

- 预招商任务列表：受ABAC策略过滤（按负责人/部门/园区）
- 一键发布操作：有载体发布权限的用户
- 配置管理：系统管理员

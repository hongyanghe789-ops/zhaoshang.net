## 一、功能概述

### 1.1 背景与目标

招商网络平台/UL App上存在大量企业发布的"选址需求"（如"需要500㎡研发办公空间，预算3元/㎡·天"）。本功能基于园区/房源属性，**自动匹配平台选址需求**，将匹配结果主动Push给园区招商人员，实现从"被动等客"到"主动出击"的转变。

### 1.2 核心价值

- 变被动为主动，提升招商效率
- 精准匹配，减少无效沟通
- 利用UL + HLLM（大语言模型）实现智能推荐

## 二、匹配引擎

### 2.1 匹配维度

| 维度 | 选址需求字段 | 园区/房源属性 | 匹配逻辑 |
|------|-------------|-------------|---------|
| 面积 | 需求面积范围 | 房源可租面积 | 面积在需求范围±20%内 |
| 价格 | 预算范围 | 房源挂牌价 | 价格在预算范围内 |
| 区域 | 意向城市/区域 | 园区所在城市/区域 | 完全匹配或临近区域 |
| 类型 | 载体类型需求 | 房源类型 | 完全匹配 |
| 行业 | 企业所属行业 | 园区主导产业（动态表单） | 产业匹配度评分 |
| 配套 | 特殊配套需求 | 园区配套信息 | 包含所需配套 |
| 入驻时间 | 期望入驻时间 | 房源可用时间 | 可用日期 ≤ 期望入驻时间 |

### 2.2 匹配分值计算

```
总匹配分 = Σ(维度匹配分 × 维度权重)

维度权重（可配置）：
  面积匹配：20%
  价格匹配：25%
  区域匹配：20%
  类型匹配：15%
  行业匹配：10%
  配套匹配：5%
  时间匹配：5%

匹配分值：
  完全匹配：100分
  部分匹配：50~99分（按偏离程度线性递减）
  不匹配：0分

推送阈值：总匹配分 ≥ 60分（可配置）
```

### 2.3 HLLM智能增强

利用大语言模型对选址需求进行语义理解：

| 场景 | LLM作用 |
|------|---------|
| 模糊需求解析 | "要离地铁近的办公室" → 提取：交通便利度=高 |
| 行业关联 | "做AI的公司" → 匹配：人工智能/大数据/软件开发 |
| 隐含需求推断 | "生物医药企业" → 推断：需要环评/排污配套 |

## 三、Push通知

### 3.1 推送方式

| 渠道 | 形式 |
|------|------|
| 站内信 | 消息卡片："有X条新的选址需求与您管理的房源匹配" |
| UL App推送 | Push通知 → 点击查看匹配详情 |
| 旗舰版工作台 | 待办事项卡片 |

### 3.2 推送卡片内容

```
┌──────────────────────────────────┐
│ 📌 新选址需求匹配                 │
│                                  │
│ 企业：某某智能科技                 │
│ 需求：300-500㎡研发办公            │
│ 预算：2.5-3.5元/㎡·天             │
│ 区域：XX区                        │
│ 匹配度：85%                       │
│                                  │
│ 匹配房源：A栋-5层-501 (420㎡)     │
│                                  │
│ [查看详情]  [发起对接]  [忽略]     │
└──────────────────────────────────┘
```

### 3.3 操作动作

| 操作 | 说明 |
|------|------|
| 查看详情 | 跳转到选址需求详情页 |
| 发起对接 | 通过UL App IM发起沟通 |
| 忽略 | 标记为已忽略，不再推送该需求 |
| 不感兴趣 | 标记为不匹配，训练匹配模型 |

## 四、匹配管理

### 4.1 匹配列表

**入口**：载体管理 → 选址匹配（新增菜单项）

**列表字段**：匹配时间、选址企业、需求摘要、匹配房源、匹配度、对接状态、操作

**筛选条件**：匹配度范围、对接状态、时间范围、园区

### 4.2 对接状态

```
[待处理] → [已发起对接] → [对接中] → [已签约] / [未成交]
  │
  └→ [已忽略]
```

## 五、系统配置

**入口**：系统设置 → 载体管理 → 选址匹配配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 推送阈值 | 匹配度≥N%才推送 | 60% |
| 每日推送上限 | 每人每天最多接收N条匹配推送 | 10条 |
| 推送时段 | 仅在工作时间推送 | 09:00~18:00 |
| 维度权重 | 各匹配维度的权重配置 | 见2.2 |
| 自动对接 | 匹配度≥90%是否自动发起IM对接 | 关闭 |

## 六、数据权限

- 匹配推送：仅推送给房源负责人/有招商权限的人员
- 匹配列表：受ABAC策略过滤
- 配置管理：系统管理员
